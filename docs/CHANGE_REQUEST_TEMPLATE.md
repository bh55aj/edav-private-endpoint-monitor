## Implementation Plan

1. Pre-run validation report generated by EDAV Resource Monitor
2. Team reports distributed to resource owners for review
3. Owners approve specific resources via Excel (ApprovedToDelete = Yes)
4. Dry-run deletion simulated with `--dry-run` flag
5. Live deletion run with `--mode delete --delete-approved` and CONFIRM prompt
6. Post-delete verification run to confirm resources are gone
7. ARM backups retained in `backups/` directory for 90 days

**CLI commands used:**

```bash
# Dry run
python main.py --mode delete --resource-type private-endpoints \
  --input approvals.xlsx --delete-approved --dry-run \
  --subscriptions "OCIO-TSBDEV-C1,OCIO-TSBPRD-C1"

# Live delete
python main.py --mode delete --resource-type private-endpoints \
  --input approvals.xlsx --delete-approved \
  --subscriptions "OCIO-TSBDEV-C1,OCIO-TSBPRD-C1"
```

## Risk

| Risk | Mitigation |
|------|-----------|
| Accidental deletion of active endpoint | 15-gate safety check blocks deletion of Connected endpoints |
| Terraform-managed resources deleted | TerraformChecker blocks all TF-managed resources |
| Production resource deleted | Production environment gate blocks prod subscriptions |
| No approval | ApprovedToDelete=Yes + ticket + approver all required |
| Irreversible deletion | ARM backup stored in `backups/` before every deletion |

**Risk Level:** LOW (all SAFE_DELETE resources validated by automated tool)

---

## Rollback

If rollback is required after deletion:

1. Locate ARM backup in `backups/privateEndpoints/<name>_<timestamp>.json`
2. Review `rollback_instructions` field in the delete report
3. Reconstruct resource using az CLI:
   ```bash
   az network private-endpoint create \
     --name <name> \
     --resource-group <rg> \
     --private-connection-resource-id <backend-id> \
     --connection-name <name>-conn
   ```
4. Re-approve private link connection if needed
5. Notify service owner team of rollback

**Rollback Time Estimate:** 30–60 minutes per endpoint
